# 中望技术社区自动签到（青龙面板版）

🤖 适配青龙面板的中望技术社区每日自动签到脚本，获取积分奖励。

## ✨ 功能特点

- ✅ **双模式运行**：API 模式（轻量快速）+ Selenium 模式（备用兼容）
- ✅ **多账号支持**：支持多账号批量签到
- ✅ **青龙面板适配**：标准青龙脚本格式，支持环境变量配置
- ✅ **青龙通知推送**：内置青龙通知模块，支持多渠道推送
- ✅ **通知级别控制**：0=关闭 1=仅异常 2=全部通知
- ✅ **调试模式**：支持 DEBUG 环境变量调试
- ✅ **自动降级**：API 模式失败自动降级到 Selenium 模式

## 📦 快速开始

### 方式一：ql repo 拉取（推荐）

在青龙面板「定时任务」中新建任务：

```
ql repo https://github.com/Hayfan-wu/QL-ZW-CHECKIN.git "zw_" "" "" "main"
```

### 方式二：手动上传

1. 下载 `zw_checkin.py` 文件
2. 上传到青龙面板的脚本目录

## ⚙️ 环境变量配置

在青龙面板「环境变量」中添加以下变量：

| 变量名 | 说明 | 必填 |
|--------|------|------|
| `ZWSOFT_USERNAME` | 中望社区账号（手机号/邮箱），多账号换行分隔 | ✅ |
| `ZWSOFT_PASSWORD` | 中望社区密码，多账号换行分隔（与账号一一对应） | ✅ |
| `ZWSOFT_NOTIFY` | 通知级别：0=关闭 1=仅异常 2=全部通知（默认1） | ❌ |
| `ZWSOFT_DEBUG` | 调试模式：true/false（默认false） | ❌ |
| `ZWSOFT_MODE` | 运行模式：api/selenium（默认api） | ❌ |

### 多账号配置示例

**ZWSOFT_USERNAME:**
```
13800138000
13900139000
```

**ZWSOFT_PASSWORD:**
```
password123
password456
```

> 💡 账号和密码按顺序一一对应，第1行账号对应第1行密码

## 🚀 运行模式说明

### API 模式（默认，推荐）

- 优点：轻量快速，无需安装浏览器，资源占用少
- 缺点：可能受 API 变更影响

环境变量设置：
```
ZWSOFT_MODE=api
```

### Selenium 模式（备用）

- 优点：模拟真实浏览器，兼容性好
- 缺点：需要安装 Chrome 浏览器和相关依赖，资源占用大

环境变量设置：
```
ZWSOFT_MODE=selenium
```

> 💡 Selenium 模式需要额外安装依赖：
> ```
> pip install selenium webdriver-manager
> ```
> 青龙面板中需要在「依赖管理」→「Python3」中添加

## 🔔 通知推送

脚本内置青龙面板通知模块，支持以下推送渠道（在青龙面板配置文件中设置）：

- Server酱
- 企业微信机器人/应用
- 钉钉机器人
- Bark
- Telegram Bot
- PushPlus
- 飞书机器人
- iGot
- WxPusher

### 通知级别控制

通过 `ZWSOFT_NOTIFY` 环境变量控制：

- `0`：关闭所有通知
- `1`（默认）：仅异常时通知（签到失败、脚本错误）
- `2`：每次运行都通知

## 📋 定时任务

在青龙面板「定时任务」中创建：

- **任务名称**：中望社区签到
- **命令**：`task zw_checkin.py`
- **定时规则**：`0 0 1 * * *`（每天凌晨1点）

## 🔧 常见问题

### Q: API 模式登录失败怎么办？

A: 尝试切换到 Selenium 模式：
```
ZWSOFT_MODE=selenium
```

### Q: 多账号怎么配置？

A: 在环境变量中每行一个账号，密码同理，顺序一一对应。

### Q: 收不到通知？

A: 1. 检查青龙面板的通知配置是否正确
   2. 确认 `ZWSOFT_NOTIFY` 不是 0
   3. 查看脚本日志中是否有通知发送失败的提示

### Q: 签到失败提示验证码？

A: 中望社区登录有滑块验证码，建议使用 API 模式。如果 API 模式不可用，可以尝试 Selenium 模式，但可能无法绕过验证码。
